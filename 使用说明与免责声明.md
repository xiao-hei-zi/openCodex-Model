# Codex Picker 动态模型同步脚本

## 一、用途

本脚本用于已经完成前端补丁的便携版 Codex 桌面程序。

启动时，脚本会读取本机 OpenCodex 生成的模型缓存，动态检测当前已经配置并可列出的提供商模型，将检测结果写入模型选择器资源，然后按需重新打包 `app.asar`。

脚本不包含固定模型白名单，也不包含用户名、API 密钥、访问令牌或个人绝对路径。

## 二、目录结构

默认情况下，将以下文件和目录放在同一级：

```text
工作目录/
├─ Launch-Codex-Picker.cmd
├─ Sync-And-Launch-Codex-Picker.ps1
├─ Config.example.ps1
├─ Config.ps1                         可选
├─ Codex-Picker/
├─ app-unpacked/
└─ tools/
```

其中：

- `Codex-Picker`：便携版 Codex 程序。
- `app-unpacked`：已经解包并完成模型菜单补丁的前端资源。
- `tools`：包含 `@electron/asar`，也可以放置 Node.js。

## 三、填写参数

默认目录结构符合要求时，不需要配置。

需要自定义路径时：

1. 复制 `Config.example.ps1`。
2. 将副本重命名为 `Config.ps1`。
3. 修改其中的 `$CodexPickerConfig`。

示例：

```powershell
$CodexPickerConfig = @{
    PortableDirectory = 'Codex-Picker'
    UnpackedDirectory = 'app-unpacked'
    AsarCliPath = 'tools\node_modules\@electron\asar\bin\asar.mjs'
    ProfileDirectory = 'profile'
    ExecutableName = 'ChatGPT.exe'
    WindowTitle = 'Codex Picker Patched'
    ModelCachePath = ''
    NodePath = ''
}
```

相对路径均以脚本所在目录为基准。也可以填写绝对路径，但不要把带有个人信息、密钥或内部服务器地址的 `Config.ps1` 公开发布。

### 参数说明

| 参数 | 说明 |
|---|---|
| `PortableDirectory` | 便携版 Codex 程序目录 |
| `UnpackedDirectory` | 解包且已打补丁的前端目录 |
| `AsarCliPath` | `asar.mjs` 的路径 |
| `ProfileDirectory` | 便携版独立 Electron 用户数据目录 |
| `ExecutableName` | Codex 桌面端可执行文件名 |
| `WindowTitle` | 补丁窗口标题 |
| `ModelCachePath` | 可选的模型缓存路径；留空时自动发现 |
| `NodePath` | 可选的 Node.js 路径；留空时自动发现 |

## 四、模型缓存发现

`ModelCachePath` 留空时，脚本依次使用：

```text
%CODEX_HOME%\models_cache.json
%USERPROFILE%\.codex\models_cache.json
```

动态模型条件：

```text
slug 包含 /
visibility 为 list 或未设置
```

模型名称和说明来自当前 OpenCodex 模型缓存，不在脚本中手工维护。

## 五、运行

双击：

```text
Launch-Codex-Picker.cmd
```

仅检查配置和模型，不修改、不打包、不停止进程、不启动程序：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass `
  -File .\Sync-And-Launch-Codex-Picker.ps1 -ValidateOnly
```

## 六、运行行为

模型清单未变化时：

```text
直接启动或复用现有便携版窗口
不重新打包
不重启进程
```

模型清单发生变化时：

```text
重新生成 opencodex-models.js
重新打包 app.asar
只停止可执行路径位于便携版目录内的进程
部署新 app.asar
启动便携版 Codex
```

脚本不会按进程名称无条件停止系统中的所有 Codex 或 ChatGPT 进程。

## 七、安全提示

- 不要将 API 密钥、访问令牌、Cookie 或账号凭据写入脚本。
- 不要公开包含个人路径或内部地址的 `Config.ps1`。
- 修改前备份原始 `app.asar` 和便携版目录。
- 仅在自己拥有或获准管理的计算机和账户上使用。
- Codex/OpenCodex/服务商更新后，旧补丁可能失效或导致程序无法启动。
- 首次使用或升级后先运行 `-ValidateOnly`。
